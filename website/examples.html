Examples


Below are some business logic examples using Mailtools


  • Using the user inbox, create a new mbox file called baz.mbox containing every mail that does not contain the string 'John' in its To field.:
    • mailtools search -t 'John' --negate --output baz.mbox

  • Using the user's inbox and the Mailscript sort_to_folders.ms, create a cronjob that performs the scripts actions every day at midnight:
    • mailtools process --file sort_to_folders.ms --cron '0 0 * * *'

  • Using the archive foo.mbox to search for the Subject 'Space'. For all matches, create a csv of each entity named bar.csv, containing the From and Date field:
    • mailtools --use foo.mbox search -s 'Space' --list_field 'From' --list_field 'Date' > bar.csv

  • Using the mbox, revenge.mbox, match all mail in it and forward it to spammer@hack.com:
    • mailtools --use revenge.mbox search --forward spammer@hack.com

  • Just like the above example, but instead of sending to just one address, pull a number of addresses from the user's inbox mailbox that were filtered out by having the word 'spam' in the email's contents:
    • mailtools search -b 'spam' --list_field 'From' | sort | uniq | paste -sd ' ' | xargs -I{} mailtools --use revenge.mbox search --forward {}
